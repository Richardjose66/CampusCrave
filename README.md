📝 Project Overview

CampusCrave is a robust, web-based database application developed during my internship to streamline internal campus operations. It replaces manual, paper-based tracking with a dynamic digital workflow, providing campus administrators and users with an intuitive interface to manage student requests, facility bookings, and inventory.

✨ Core Features

Role-Based Access Control: Secure login environments tailored for different user types (e.g., Students, Administrators, Staff).

Interactive Data Grids: Allows users to easily filter, sort, and update campus records in real-time.

Custom Dashboards: Provides visual analytics and high-level summaries of daily order volumes, popular menu items, and active meal plan statuses.

Automated Workflows: Utilizes PL/SQL processes to automate order status updates, inventory deductions, and confirmation notifications.


🛠️ Technical Architecture

Frontend: Oracle APEX Universal Theme, HTML5, Custom CSS and JavaScript for UI enhancements.

Backend: Oracle Database, PL/SQL for complex business logic and page processes.

Database Design: Relational schema optimized for fast querying and data integrity.

🚀 How to Run Locally

To explore the application structure and UI:

Download the f257491.sql export file from this repository.

Log into your Oracle APEX Workspace.

Navigate to App Builder > Import.

Upload the file and follow the installation wizard to deploy the application into your workspace.
